# Agent Instructions

## Project Overview

이 저장소는 아직 초기 상태입니다. 아래 항목은 프로젝트가 커지면 채워 넣는 기본 템플릿입니다.

## Working Rules

- 기술 스펙과 예시는 기본적으로 Python 기준으로 작성합니다.
- 백엔드는 `FastAPI` 기준으로 작성합니다.
- 데이터베이스는 기본적으로 메모리 저장 방식을 사용합니다.
- 프론트엔드는 `React` 기준으로 작성합니다.
- 기존 파일을 우선 확인하고, 불필요한 새 파일 생성을 피합니다.
- 수정은 최소 범위로 진행합니다.
- 코드 변경 후에는 가능한 범위에서 검증합니다.
- 사용자 요청이 모호하면, 합리적인 기본값으로 진행하되 필요한 경우만 확인 질문을 합니다.

## File Conventions

- 새 파일은 ASCII를 기본으로 작성합니다.
- 마크다운은 짧은 제목과 명확한 목록 중심으로 작성합니다.
- 설정값은 예시가 아니라 실제 사용 가능한 기본값을 우선으로 둡니다.
- 기본적인 변수는 타입 변수를 명시합니다. 예를 들어 Python 코드에서는 `name: str`, `count: int`처럼 작성합니다.

## Recommended Structure

- `src/`: 애플리케이션 소스
- `tests/`: 테스트 코드
- `docs/`: 문서
- `config/`: 환경별 설정

## Notes

- 프로젝트에 맞는 실행 명령, 린트, 테스트, 빌드 명령은 여기에 추가합니다.
- 팀 규칙이나 배포 규칙이 생기면 이 파일을 기준 문서로 유지합니다.
